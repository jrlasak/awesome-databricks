# Contributing to Awesome Databricks

Thank you for helping make this list better! Here's how to contribute.

## Guidelines

- **Search for duplicates** before submitting a new resource.
- **One suggestion per pull request** - makes review faster.
- **Databricks-specific or strongly Databricks-integrated** - generic data tools don't qualify.
- **Active and maintained** - last update within 12 months (or stable/complete for books and courses).

## Quality Criteria

| Type | Minimum Bar |
|------|------------|
| **Tools/Libraries** | 50+ GitHub stars OR active production use |
| **Creators** | 5K+ followers, posts Databricks content regularly (not just bio mention) |
| **Courses** | 4+ star rating OR from a recognized platform (Databricks Academy, Microsoft Learn, etc.) |
| **Books** | Published within the last 3 years |
| **Blogs** | Active in the last 6 months |

## Link Format

Follow this format for tools and libraries:

```
- [Name](URL) - Brief description of what it does. By Author/Company.
```

Keep descriptions factual and concise. No marketing copy.

## How to Submit

1. Fork this repository
2. Add your resource to the bottom of the relevant category
3. Make sure the link works and the description is accurate
4. Submit a pull request with a clear title (e.g., "Add spark-testing-base to Testing & Quality")

## New Categories

If your resource doesn't fit an existing category, open an issue first to discuss adding a new one.

## Reporting Issues

Found a broken link, outdated info, or deprecated tool? Open an issue or submit a PR with the fix. We mark deprecated tools explicitly - that's a feature, not a bug.
